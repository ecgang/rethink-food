## What it does

### Command Center (`/`) — the daily operating view
- **Meal lifecycle** funnel: planned → produced → delivered → verified.
- **Unit economics**: line-itemed cost, reimbursement revenue, and **contribution margin sliced by program, kitchen, restaurant, contract, or market**.
- **"Act on today"** — a deterministic exception engine that surfaces what's wrong _right now_ (meals stuck before delivery, deliveries unverified, a kitchen over food budget, a contract billing window overdue), each with a severity and a recommended action.
- **Medically Tailored Meals** program health: retention, delivered-vs-prescribed, Social Care Network attribution.

### The AI operating layer — human-reviewed, grounded, fallback-safe

> The deterministic engines own every number. **AI only narrates, drafts, and retrieves — and a human approves anything that leaves the system.** This is enforced *structurally*: the AI modules can't import the database, so a generator is incapable of computing a figure. Every feature degrades to a deterministic path when no API key is set.

- **Ask the Operating Layer (`/ask`)** — natural-language Q&A over partners, funders, and contracts via **agentic Prisma retrieval (no embeddings)**; every answer **cites the real records** it used and links back to them.
- **Today's briefing** (dashboard) — narrates the exception engine into a plain-English morning briefing; an anti-hallucination guard drops anything the model invents and keeps the engine's severities.
- **Draft-and-approve comms (`/drafts`)** — drafts partner clarifications, delivery nudges, reconciliation notes, and a board narrative into a `DRAFT → APPROVED / DISCARDED` queue. **Nothing is ever auto-sent.**
- **AI intake (`/intake`)** — a free-text partner email becomes a structured request with **per-field confidence**; an operator approves before anything is written; every decision hits the audit trail.

### Network & marketplace
- **Partners / Funders** — a real NYC partner directory and per-funder impact reporting ("what your support made possible") with CSV export.
- **Demand Map (`/map`)** — food-insecurity-weighted meal demand vs. fulfilled capacity by neighborhood; click a neighborhood to match a kitchen's spare capacity into scheduled meals (the marketplace loop).

### Field operator PWA (`/field`)
Installable on a phone: tap a delivery, snap a proof photo, mark delivered, verify receipt — closing the produced→delivered→verified loop from the field. Each action clears the matching "act on today" exception live.

### Operations & trust
- **RBAC** — Operations / Finance / Executive, gated by capabilities (`can()`) enforced server-side on every write; the role cookie is **HMAC-signed** (tamper-evident).
- **Reports** (`/reports`) — weekly funder/board snapshots generated by a Vercel cron. **Audit trail** (`/audit`) — every operator action, attributed and timestamped.

## Quick start

> [!IMPORTANT]
> Requires **Node 20+** and **Docker** (for local Postgres). The AI layer is optional — without `ANTHROPIC_API_KEY` every AI feature falls back to a deterministic path, so the app runs fully offline.

```bash
# 1. Postgres (Docker, on port 5433)
docker run -d --name rethink-pg -e POSTGRES_PASSWORD=rethink -e POSTGRES_DB=rethink \
  -p 5433:5432 postgres:16

# 2. Env
cp .env.example .env
#   DATABASE_URL is preset for the container above.
#   Optional: add ANTHROPIC_API_KEY to use the live model across the AI layer.

# 3. Install, migrate, seed (real-data snapshots are committed under data/)
npm install
npx prisma migrate deploy   # apply migrations (migrate dev is interactive/gated in this env)
npm run db:seed             # seeds from data/ snapshots + planted anomalies
# optional: refresh the real NYC data snapshots
npm run ingest

# 4. Dev
npm run dev                 # http://localhost:3000
```

## Verify

```bash
npm run typecheck   # tsc --noEmit
npm test            # Vitest — 350 tests (margin, exceptions, field, safety, incidents, intake evals, ask, briefing, comms…)
npm run build       # prisma generate && next build
```

**CI** (`.github/workflows/ci.yml`, Node 20) runs `npm ci` → `prisma generate` → typecheck → test → build on every push — the same four gates kept green throughout.

## Why this slice (and what I deliberately cut)

The posting screens for the judgment to *"distinguish an essential workflow from an impressive but unnecessary feature"* and a *"bias toward shipping rather than overengineering."* So this goes **deep** on the flagship operating system, a full **human-reviewed AI operating layer**, the **network + marketplace loop**, and an installable **field app** — and **cuts the rest on purpose**:

- ❌ **Full SSO / identity provider** — capabilities are enforced server-side and the role cookie is HMAC-signed; only role *selection* is open (no login wall), by demo choice, so you can switch roles and watch the permission model work. SSO swaps in behind the same `can()` checks.
- ❌ **An ML anomaly model** — one set of transparent, tunable thresholds is more trustworthy than a half-trained black box, and an operator can argue with it.
- ❌ **Source reconciliation** (HubSpot/CSV → canonical schema) and an **offline write-queue** for the field app — real production work, not the thesis.
- ❌ **Infra beyond serverless Postgres + Vercel** — a disciplined build-vs-buy call.

The cutting *is* the signal. Every non-goal is documented in [`docs/DECISIONS.md`](docs/DECISIONS.md).

## Grounded in real NYC data

Not seeded with invented places — an ingestion pipeline (`npm run ingest`, scripts in `scripts/ingest/`) pulls real NYC open data, normalizes it, and commits deterministic, offline-safe snapshots to `data/`:

- **Neighborhoods + coordinates** — NYC Open Data *2020 Neighborhood Tabulation Areas* (`9nt8-h7nd`); centroids computed per NTA across all five boroughs.
- **Partner network** — Rethink Food's **actual published partners**: 35 restaurant partners (incl. Rethink Certified — Manna's, Rob's Kitchen, Sophie's Cuban…) and 52 community-based orgs (Henry Street Settlement, The Bowery Mission, North Brooklyn Angels…), geocoded via the US Census geocoder.
- **Demand** — weighted by *Feeding America Map the Meal Gap* county food-insecurity rates.
- **Social Care Networks** — the real NY 1115-waiver leads by borough: **PHS** (Manhattan/Brooklyn/Queens), **SOMOS** (Bronx), **SIPPS** (Staten Island).

## Trust, quality & accessibility

The hardest part of an operating system isn't the charts — it's that everyone believes the numbers.

- **One source of truth for metrics.** `lib/definitions.ts` defines meal / realized / cost / revenue / contribution-margin once; every view derives from the same pure functions, surfaced in-app via the "How these numbers are defined" panel.
- **Numbers that provably reconcile.** `tests/metrics.test.ts` asserts the sum of *every* slice equals the headline total — so no two views can silently disagree.
- **AI that can't fabricate numbers.** The AI modules can't reach the database; the briefing's `filterToKnown` guard rejects any exception the engine didn't emit; Ask answers cite real record ids.
- **Accessibility** — WCAG 2.2 AA: AA-contrast tokens, visible focus rings, `role="img"` + labels on charts, a keyboard-accessible map equivalent, `aria-live` on AI results, `prefers-reduced-motion`. Responsive to mobile.

## Deploy

Built for **Vercel + Neon** (serverless Postgres). Set `DATABASE_URL`, `ANTHROPIC_API_KEY`, and `ROLE_COOKIE_SECRET` in the Vercel project; apply migrations to Neon with `prisma migrate deploy` (using the unpooled connection) and seed with `npm run db:seed`. `prisma generate` runs automatically on install/build.
