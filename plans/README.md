## Findings considered and rejected (do not re-audit)

- **CSRF on server actions** — by design mitigated; Next.js verifies the Origin
  header on all Server Actions. Not a finding.
- **SQL injection** — none; Prisma is parameterized and inputs are Zod-validated.
  (A strength of the codebase.)
- **No authentication / spoofable role cookie** — intentional per ADR 10
  (lightweight role signer, no login wall). Known limitation, not a defect.
- **postcss moderate advisory (transitive via `next`)** — real but low impact
  (CSS is build-time and trusted); the only remediation bumps `next`, which is
  riskier than the issue. Revisit on the next planned `next` upgrade.
- **Split the 484-line `lib/queries.ts` into modules** — borderline; defensible
  at this size and high churn risk. Not worth it now.
- **React / `@types/node` minor version lag; intake `rawInput` render (React
  escapes); assorted type-hygiene nits (`as` casts, non-null assertions)** — low
  leverage; left as-is.
- **Prisma v7 upgrade** — explicitly out of scope; v6 is pinned by ADR 8.
